# Database/File Documentation

This project does not use a traditional database. Instead, it uses CSV and JSON files as the data exchange layer between the FastAPI backend, the C17 matching engine, and the React dashboard.

## File Summary

| File | Type | Role |
| --- | --- | --- |
| `engine/data/orders_large.csv` | CSV | Input orders for the C matching engine |
| `engine/data/trades.csv` | CSV | Executed simulated trade history |
| `engine/benchmark/benchmark.csv` | CSV | Engine performance benchmark metrics |
| `engine/data/order_book_snapshot.json` | JSON | Final buy/sell order book snapshot |

## engine/data/orders_large.csv

### Producer

- FastAPI backend through `POST /generate-market-orders`
- Python generator script `engine/benchmark/generate_orders.py`

### Consumer

- C17 matching engine
- Specifically loaded by `engine/src/csv_loader.c`
- Used when `engine/src/main.c` runs the engine

### Format

CSV with header:

```csv
order_id,side,price,quantity
```

Example:

```csv
order_id,side,price,quantity
1,B,201.10,500
2,S,201.75,250
```

Field description:

| Field | Type | Description |
| --- | --- | --- |
| `order_id` | integer | Unique order identifier |
| `side` | char | `B` for buy order, `S` for sell order |
| `price` | decimal | Limit price for the order |
| `quantity` | integer | Number of units in the order |

### Purpose

This file is the main input file for the matching engine. It stores simulated buy and sell orders. In the market-based simulation workflow, the backend generates this file around the live Finnhub reference price.

## engine/data/trades.csv

### Producer

- C17 matching engine
- Trade logging logic in `engine/src/trade.c`

### Consumer

- FastAPI backend through `GET /trades`
- React frontend trade log panel

### Format

CSV with header:

```csv
trade_id,buy_order_id,sell_order_id,price,quantity
```

Example:

```csv
trade_id,buy_order_id,sell_order_id,price,quantity
1,10,11,100.00,50
2,12,13,100.00,10
```

Field description:

| Field | Type | Description |
| --- | --- | --- |
| `trade_id` | integer | Unique executed trade identifier |
| `buy_order_id` | integer | Matched buy order ID |
| `sell_order_id` | integer | Matched sell order ID |
| `price` | decimal | Execution price |
| `quantity` | integer | Executed quantity |

### Purpose

This file records all simulated trades generated by the matching engine. The dashboard reads this data through the backend and displays it in the trade log.

## engine/benchmark/benchmark.csv

### Producer

- C17 matching engine
- Benchmark logic in `engine/src/benchmark.c`

### Consumer

- FastAPI backend through `GET /benchmark`
- React frontend KPI cards and benchmark history panel

### Format

CSV with header:

```csv
orders_processed,execution_time_ms,throughput_ops_sec,avg_latency_us
```

Example:

```csv
orders_processed,execution_time_ms,throughput_ops_sec,avg_latency_us
10000,86.000,116279.070,8.600
```

Field description:

| Field | Type | Description |
| --- | --- | --- |
| `orders_processed` | integer | Number of orders processed by the engine |
| `execution_time_ms` | decimal | Total execution time in milliseconds |
| `throughput_ops_sec` | decimal | Orders processed per second |
| `avg_latency_us` | decimal | Average latency per order in microseconds |

### Purpose

This file stores performance results from the latest engine run. It is used to evaluate matching engine speed, throughput, and latency.

## engine/data/order_book_snapshot.json

### Producer

- C17 matching engine
- Order book snapshot logic in `engine/src/order_book.c`

### Consumer

- FastAPI backend through `GET /order-book`
- React frontend order book panel

### Format

JSON object containing buy/sell counts and order arrays.

Example:

```json
{
  "buy_count": 2509,
  "sell_count": 2574,
  "buy_orders": [
    {
      "order_id": 7652,
      "price": 99.89,
      "quantity": 64
    }
  ],
  "sell_orders": [
    {
      "order_id": 3637,
      "price": 99.9,
      "quantity": 74
    }
  ]
}
```

Field description:

| Field | Type | Description |
| --- | --- | --- |
| `buy_count` | integer | Number of remaining buy orders |
| `sell_count` | integer | Number of remaining sell orders |
| `buy_orders` | array | Remaining buy-side order book rows |
| `sell_orders` | array | Remaining sell-side order book rows |
| `order_id` | integer | Order identifier |
| `price` | decimal | Order limit price |
| `quantity` | integer | Remaining order quantity |

### Purpose

This file stores the final order book state after matching is complete. The dashboard uses it to display remaining buy and sell orders, best bid, best ask, and spread.

## File-Based Data Flow

```text
Live Finnhub Price
    |
    v
FastAPI /generate-market-orders
    |
    v
engine/data/orders_large.csv
    |
    v
C17 Matching Engine
    |
    +--> engine/data/trades.csv
    +--> engine/benchmark/benchmark.csv
    +--> engine/data/order_book_snapshot.json
    |
    v
FastAPI Result Endpoints
    |
    v
React Dashboard
```
